# Kizamu（刻む）- 基本設計書

## 機能一覧（MVP）

### 1. デイリーチェックイン（毎日の記録）

**概要**
1日1回、その日を振り返る記録を残す。低摩擦・短時間で完結する設計。

**フロー**
1. アプリを開く → 「今日の振り返り」画面（当日分が未記録なら自動表示）
2. 以下の項目を入力（全て任意）
   - 今日やったこと（自由テキスト or タグ選択）
   - 今日の気分（5段階 or 絵文字）
   - 小さな達成（簡単な一言）
   - 明日やりたいこと（任意）
3. 「記録する」ボタンで保存

**設計上のポイント**
- 入力は最短30秒で完了できる量にとどめる
- 「今日は特になし」ボタンも用意し、記録すること自体に価値を置く
- 夜にプッシュ通知（時刻は設定可能）

---

### 2. マイクロ達成の記録

**概要**
小さなことでも「達成」として1日中いつでも追記できる。

**フロー**
- ウィジェットやクイックアクションから、一言メモを投下
- 例：「散歩した」「野菜食べた」「返信できた」
- タグ付け可能（健康・人間関係・仕事・学び・趣味 など）

**データ構造（論理）**
```
MicroAchievement {
  id: UUID
  date: Date
  text: String
  tags: Tag[]
  createdAt: Timestamp
}
```

---

### 3. やりたいことリスト（Bucket List）

**概要**
人生でやりたいこと・経験したいことを一覧で管理する。
「夢リスト」として登録しておき、日々の記録と紐づけて進捗を残す。

**アイテムの構造**
- タイトル（例：「富士山に登る」「小説を書き上げる」「30ヶ国旅行する」）
- カテゴリ（旅・創作・学び・健康・人間関係・挑戦 など）
- ステータス：`dream`（夢）→ `inProgress`（動き始めた）→ `done`（達成）
- メモ・詳細（任意）
- 関連する Life Orientation（後述）への紐づけ（任意）

**詳細画面**
- このアイテムに紐づいた過去の日々の記録を時系列で表示
- 「最後に動いたのは〇日前」を表示
- 進捗メモを自由に追記できる（マイルストーン記録）

**チェックイン時の紐づけ**
- 達成記録を保存する際に「これはやりたいことリストの何かに関係しますか？」と任意で問いかけ
- 例：「登山の練習で近所の山を歩いた」→「富士山に登る」に紐づける

---

### 4. 人生の方向性（Life Orientation）

**概要**
具体的な「やりたいこと」とは別に、「こう生きたい」という価値観・方向性を登録する。
Bucket List や日々の記録の文脈を与える背景軸として機能する。

**例**
- 「健康に長く生きたい」
- 「創作活動を一生続けたい」
- 「大切な人との時間を最優先にしたい」

期限なし・達成/未達成でなく、**ずっと向き合い続けるもの**として扱う。

---

### 5. 四半期マンダラ（Quarterly Mandala）

**概要**
大谷翔平が使ったことでも知られる曼荼羅チャートを四半期単位の計画に応用する。
3×3 のグリッドで、中央に四半期テーマを置き、周囲8マスに注力項目を設定する。

**グリッド構成**

```
┌──────────┬──────────┬──────────┐
│  注力 ①  │  注力 ②  │  注力 ③  │
├──────────┼──────────┼──────────┤
│  注力 ④  │ 四半期   │  注力 ⑤  │
│          │ テーマ   │          │
├──────────┼──────────┼──────────┤
│  注力 ⑥  │  注力 ⑦  │  注力 ⑧  │
└──────────┴──────────┴──────────┘
```

- **中央マス**：四半期のテーマ（例：「自分の土台をつくる」「仕事に集中する3ヶ月」）
- **周囲8マス**：Bucket List や Life Orientation から選んでピックアップ（空きマスがあっても良い）

**四半期の定義**
- Q1: 1〜3月 / Q2: 4〜6月 / Q3: 7〜9月 / Q4: 10〜12月
- 手動で任意の期間を設定する形も可

**マンダラとの日々の連動**
- ホーム画面に「今四半期のフォーカス」としてマンダラのサマリーを表示
- 日々のチェックイン時、「今日の行動はマンダラの何かに関係しますか？」と問いかけ（任意）
- 各マスから「このテーマに関連した記録一覧」を参照できる
- 四半期末に「振り返りモード」が起動し、各マスに対してどれだけ動けたかをレビューできる

**マスの詳細**

| 項目 | 内容 |
|------|------|
| タイトル | 注力項目の名前 |
| 紐づき | Bucket List アイテム or Life Orientation（任意） |
| 関連記録数 | この四半期に紐づいた記録の件数 |
| メモ | このマスに対するメモ・意気込み |

---

### 6. 振り返りカレンダー・ライフグラフ

**概要**
過去の記録を可視化し、「自分はちゃんと生きていた」ことを実感できる画面。

**表示パターン**
- **カレンダービュー**：月ごとに記録があった日を色付き表示（GitHub の草のようなイメージ）
- **ライフグラフ**：気分の推移・達成数のグラフ（週・月単位）
- **ハイライトリール**：「先月のあなた」として過去の達成を自動サマリー表示

**色分けの例**
- 記録なし：グレー
- 記録あり：薄いグリーン
- 達成あり：グリーン
- 気分が良かった日：ゴールド

---

### 7. ストリーク（継続記録）— 責めない設計

**概要**
継続を称えるが、途切れを責めない仕組み。

**仕様**
- 連続記録日数を表示（励まし目的）
- ストリークが途切れても「過去の記録は消えない」ことを明示
- 「リカバリーモード」：3日以上空いても「おかえり」メッセージで迎える
- 累積記録日数（通算）を主要指標として前面に出す

## 画面設計（画面一覧）

| 画面名 | 役割 |
|--------|------|
| ホーム | 今日のチェックイン促進 + 今四半期マンダラサマリー + ストリーク |
| チェックイン入力 | 今日の振り返り入力 + Bucket List / マンダラへの紐づけ |
| 達成クイック入力 | マイクロ達成を素早く記録するモーダル |
| やりたいことリスト | Bucket List の一覧（カテゴリ・ステータスでフィルター） |
| やりたいこと詳細 | 個別アイテム + 関連記録一覧 + 進捗メモ |

| マンダラ振り返り | 四半期末のレビュー画面（各マスへの動きを確認） |

| カレンダー | 月次カレンダー（Bucket List / マンダラ別フィルター） |
| ライフグラフ | 気分・達成数・テーマ別行動数の推移 |
| 設定 | 通知時刻、四半期の開始月、テーマ、タグ管理など |

## データモデル（論理設計）

```
User {
  id: UUID
  name: String
  createdAt: Timestamp
  settings: UserSettings
}

DailyEntry {
  id: UUID
  userId: UUID
  date: Date                  // 1日1件
  mood: Int(1-5)              // 気分
  summary: String             // 今日のまとめ（自由記述）
  tomorrowNote: String        // 明日やりたいこと
  achievements: Achievement[] // その日の達成リスト
  goalLinks: GoalLink[]       // 紐付いた目標
  createdAt: Timestamp
  updatedAt: Timestamp
}

Achievement {
  id: UUID
  entryId: UUID
  text: String
  tags: Tag[]
  isQuick: Boolean            // クイック入力かどうか
  createdAt: Timestamp
}

// 人生の方向性（価値観・軸）
LifeOrientation {
  id: UUID
  userId: UUID
  title: String                 // 例：「健康に長く生きたい」
  description: String?
  createdAt: Timestamp
}

// やりたいことリスト
BucketItem {
  id: UUID
  userId: UUID
  title: String                 // 例：「富士山に登る」
  description: String?
  category: String              // 旅・創作・学び・健康・人間関係・挑戦 など
  status: Enum(dream, inProgress, done)
  doneAt: Date?
  orientationId: UUID?          // 紐づく LifeOrientation（任意）
  createdAt: Timestamp
}

// 四半期マンダラ
QuarterlyMandala {
  id: UUID
  userId: UUID
  year: Int
  quarter: Int(1-4)
  centerTheme: String           // 四半期テーマ（中央マス）
  cells: MandalaCell[8]         // 周囲8マス
  createdAt: Timestamp
}

MandalaCell {
  id: UUID
  mandalaId: UUID
  position: Int(1-8)            // グリッド上の位置
  title: String                 // 注力項目の名前
  bucketItemId: UUID?           // 紐づく BucketItem（任意）
  orientationId: UUID?          // 紐づく LifeOrientation（任意）
  memo: String?
}

// 達成記録 → Bucket List / マンダラへの紐づけ
RecordLink {
  id: UUID
  achievementId: UUID           // 紐づく Achievement
  bucketItemId: UUID?           // 紐づく BucketItem（任意）
  mandalaCell: UUID?            // 紐づく MandalaCell（任意）
  orientationId: UUID?          // 紐づく LifeOrientation（任意）
  note: String?
  createdAt: Timestamp
}
```

## 技術スタック

### アーキテクチャ概要

Next.js がフロントエンドとバックエンド（API）を兼ねる構成。
別途バックエンドサーバーは不要。Docker Compose でローカル完結。

```
┌─────────────────────────────┐
│        Next.js              │
│  ┌─────────┐ ┌───────────┐  │
│  │  React  │ │API Routes │  │
│  │ （画面） │ │（DBアクセス│  │
│  │         │ │  など）   │  │
│  └─────────┘ └─────┬─────┘  │
└────────────────────┼────────┘
                     │ Prisma（ORM）
              ┌──────▼──────┐
              │ PostgreSQL  │
              └─────────────┘
```

### 技術選定

| レイヤー | 採用 | 理由 |
|----------|------|------|
| フレームワーク | **Next.js 14（App Router）** | React ベース、API Routes でバックエンドも担える |
| 言語 | **TypeScript** | 型安全、長期メンテに有利 |
| スタイリング | **Tailwind CSS** | 開発速度が高い |
| ORM | **Prisma** | 型安全なDB操作、マイグレーション管理が楽 |
| データベース | **PostgreSQL** | 実績・信頼性が高い |
| 認証 | **Auth.js（NextAuth v5）** | Next.js との相性が良い、メール/OAuth 対応 |
| インフラ | **Docker Compose** | ローカルで app + db を一括起動 |

### Docker Compose 構成

```yaml
services:
  app:          # Next.js（ポート 3000）
  db:           # PostgreSQL（ポート 5432）
```

`docker compose up` 1コマンドで全て起動。

### 将来デプロイする場合

| サービス | 用途 |
|----------|------|
| Vercel | Next.js のホスティング（無料枠あり） |
| Supabase / Railway / Render | PostgreSQL のホスティング |

ローカルと同じ PostgreSQL を使うので移行コストは低い。

## 非機能要件

| 項目 | 目標値 |
|------|--------|
| チェックイン完了までの操作数 | 3タップ以内 |
| アプリ起動〜記録完了 | 1分以内 |
| オフライン対応 | 記録の読み書きはオフラインでも可能 |
| データの永続性 | ユーザーデータは削除操作なしでは消えない |
| プライバシー | 記録はデフォルト非公開、外部共有は明示的な操作のみ |
